# Poké-Data Explorer | High-Performance API Integration

## Project Overview
This is a functional web application that interfaces with the **RESTful PokéAPI** to retrieve, filter, and visualize complex datasets. Beyond a simple display, the project focuses on efficient data fetching, state management, and user experience (UX) through dynamic interface adjustments.

Live Demo: [https://alejandrocis12.github.io/Pokedex/](https://alejandrocis12.github.io/Pokedex/)

## Technical Challenges & Mathematical Logic
As an **Applied Mathematician**, I utilized this project to explore:

* **Asynchronous Data Handling:** Managed multiple API calls using `Fetch` and `Async/Await` patterns, ensuring that the UI remains responsive while processing large JSON objects.
* **Search Algorithms:** Implemented a real-time filtering system that scans the dataset to match user input, demonstrating an understanding of search complexity.
* **Dynamic Routing & State:** Created individual detail pages for over 1000 entities, utilizing URL parameters to fetch and render specific metadata dynamically.
* **UI/UX Logic:** Developed a "Dark Mode" feature using CSS variables and JavaScript state persistence (Local Storage), showing attention to accessibility and user preferences.

## 🧪 Future Roadmap: Data Analysis (In Progress)
I am currently working on a **Statistical Module** for this platform to analyze:
- **Type Distribution:** Visualizing the frequency of Pokémon types using descriptive statistics.
- **Base Stat Correlations:** Identifying mathematical relationships between different attributes (e.g., Speed vs. Defense) across generations.

## Tech Stack
* **Frontend:** HTML5, CSS3 (Custom Properties for Dark Mode).
* **Core Logic:** Vanilla JavaScript (ES6+), Fetch API, Promises.
* **Data Source:** PokéAPI (JSON-based REST API).
